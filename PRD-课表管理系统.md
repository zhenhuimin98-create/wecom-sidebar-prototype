## 一、产品概述

### 1.1 产品定位

面向教务运营人员的**主讲老师课表查看与管理工具**，用于查看师资排课情况、省心约规则执行状态、课节冲突检测等，辅助排课决策。

### 1.2 核心使用场景

| 场景 | 角色 | 目标 |
|------|------|------|
| 查看某位老师本周排课密度 | 教务运营 | 评估师资利用率，发现空闲/过满时段 |
| 检查省心约规则是否正常执行 | 教务运营 | 发现待约课/冲突时段，及时介入处理 |
| 按学员维度定位课节 | 教务/班主任 | 快速找到某学员在某老师名下的排课情况 |
| 月度排课总览 | 教务主管 | 宏观了解师资月度排课分布 |
| 省心约冲突排查 | 教务运营 | 发现其他学员占用省心约固定时段的情况 |

### 1.3 用户角色

- **教务运营**：日常排课管理、冲突处理、省心约规则监控
- **班主任/二讲老师**：查看自己关联学员的课节
- **教务主管**：月度维度审视排课情况

## 三、页面布局与全局组件

### 3.1 顶部搜索栏

水平排列，包含以下组件：

| 组件 | 类型 | 说明 |
|------|------|------|
| 页面标题 | 文本 | "课表管理"，加粗16px |
| 师资姓名/ID | 输入框 + 下拉联想 | 输入时实时过滤师资列表，选中后切换当前老师并刷新课表 |
| 学员姓名/ID/手机号 | 普通输入框 | 支持模糊匹配学员姓名、学员ID、手机号 |
| 搜索 | 蓝色按钮 | 点击后对当前课表数据执行筛选 |
| 重置 | 灰色按钮 | 清空所有搜索条件，恢复原始展示 |

#### 3.1.1 师资搜索下拉联想逻辑

```
触发时机：输入框获取焦点时 or 输入内容变化时
匹配规则：
  - 匹配师资姓名（模糊匹配，忽略大小写）
  - 匹配师资ID（前缀/包含匹配）
  - 匹配格式化标签 "GTH-{姓名}/{ID}"
展示格式：GTH-张明远/175691240266142
选中行为：
  1. 输入框填入选中项的格式化标签
  2. 关闭下拉列表
  3. 切换 selectedTeacherKey
  4. 立即触发课表重新渲染
键盘支持：Enter 键自动选中第一个匹配项
无匹配：显示"无匹配师资"灰色提示
当前选中项：下拉列表中高亮显示（蓝色背景）
```

#### 3.1.2 学员搜索筛选逻辑

```
搜索目标：当前老师名下的所有课节(lessons)
匹配字段（OR关系）：
  - lesson.studentId：包含匹配
  - lesson.studentName：包含匹配（忽略大小写）
触发方式：点击"搜索"按钮 或 输入框内按 Enter
```

#### 3.1.3 搜索结果展示

搜索生效后：
1. **筛选条件栏**出现在搜索栏下方，展示筛选标签（如"学员：李小雨"），标签可单独关闭
2. **匹配数**：右侧显示"共匹配 X 节课"
3. **课表高亮**：
   - 匹配课节：正常展示 + 蓝色发光脉冲动画（2秒后消失）
   - 非匹配课节：透明度降为 0.15（灰显）
   - 非课节元素（固定时间、省心约待约课等）：同样灰显

## 六、课表主体 — 时间轴公共规则

### 6.1 时间范围

- 起始时间：07:00
- 结束时间：24:00
- 最小粒度：30分钟（一个时间槽 = 30min）
- 总时间槽数：34 个

### 6.2 星期映射

| 值 | 含义 | 索引基准 |
|----|------|----------|
| 0 | 周一 | 一周起始日 |
| 1 | 周二 | |
| 2 | 周三 | |
| 3 | 周四 | |
| 4 | 周五 | |
| 5 | 周六 | |
| 6 | 周日 | |

### 6.3 卡片定位算法

所有视图的时间卡片均基于 **绝对定位** 渲染：

```
top = slotIndex × cellHeight (px)
height = (endSlot - startSlot) × cellHeight - 2 (px)
```

其中：
- `slotIndex = (hour - 7) × 2 + (minute >= 30 ? 1 : 0)`
- 周视图 `cellHeight = 50px`
- 日视图 `cellHeight = 60px`

## 七、月视图

### 7.1 布局

标准日历网格，7列（周一至周日），行数根据当月天数和起始星期动态计算。每个格子高度固定120px。

### 7.2 格子内容

每个日期格子包含：

1. **日期数字**（左上角，13px加粗）
   - 当日高亮蓝色
   - 非当月日期灰色（#ccc）
2. **课节/待约摘要列表**（最多显示3条）
   - 已约课节：红色圆点 + `{开始时间} {学员姓名}`
   - 省心约待约：绿色圆点 + `{开始时间} {学员姓名} 待约`
   - 优先展示已约课节，剩余名额展示待约课
3. **更多入口**：超过3条时显示 `+N 更多`，点击跳转到该日的日视图

### 7.3 月视图交互

| 操作 | 行为 |
|------|------|
| 点击课节摘要行 | 弹出课节详情弹窗 |
| 点击"+N 更多" | 跳转到对应日期的日视图 |

### 7.4 搜索状态下的月视图

- 匹配课节：正常透明度
- 非匹配课节：透明度 0.2（比周/日视图的 0.15 略高，适配小格子阅读）

## 八、周视图（默认视图）

### 8.1 布局

| 区域 | 说明 |
|------|------|
| 表头 | 左侧"时间"列（56px宽） + 7天列。每列显示日期数字+星期 |
| 表体 | 34行 × 8列（时间列 + 7天列），每行高50px |
| 时间标签 | 整点行左侧显示 "HH:00"，半点行无标签 |
| 半点行 | 底部边框改为虚线，区分整点/半点 |
| 今日列 | 表头日期和星期蓝色高亮 |

### 8.2 渲染层级与元素类型

周视图中每天的内容渲染在一个绝对定位容器（wrap）内，按以下层级叠加：

#### 第1层：师资固定时间背景（z-index: 0）

```
样式：
  - 浅蓝色背景 #f0f7ff
  - 左侧3px蓝色边线 #91caff
渲染逻辑：
  遍历 teacher.fixed 中 day 匹配的所有时间段
  每个固定时间段渲染一个独立的背景块
  多个不连续的固定时间段分别渲染（如14:00-16:00 和 17:00-21:00 分开展示）
```

#### 第2层：可预约时间 + 省心约待约卡片（z-index: 2）

在每个固定时间段内，逐槽位扫描：

```
for each slot in [fixedStart, fixedEnd):
    if slot被课节占用:
        跳过
    else if slot对应省心约规则:
        渲染【省心约待约卡片】（绿色虚线边框）
    else:
        向后合并连续空闲槽位（遇到占用/省心约规则时停止合并）
        渲染【可被预约时间卡片】（蓝色）
```

#### 第3层：非固定时间内的省心约待约卡片

```
额外检查：省心约规则时段不在任何固定时间范围内
条件：rule.day === d && 不在fixedSlots内 && 不被课节占用
渲染为【省心约待约卡片】（与第2层样式一致）
```

#### 第4层：已约课节卡片（z-index: 2）

```
遍历 teacher.lessons 中 day 匹配的课节
每节课渲染一个独立卡片
卡片上可能叠加：省心约标签、冲突标记
```

### 8.3 卡片样式定义

#### 8.3.1 可被预约时间卡片

```
类型：fixed
背景：#f0f7ff
圆点：#1677ff（蓝色）
时间文字：#1677ff
描述文字：#69b1ff "可被预约时间"
交互：不可点击
```

#### 8.3.2 已约课节卡片（省心约/OES统一样式）

```
类型：sxy 或 oes（样式统一）
背景：#fff1f0（浅红色）
圆点：#ff4d4f（红色）
时间文字：#cf1322
描述文字：#ff7875，显示 className
交互：点击弹出课节详情弹窗
右上角标记：
  - 省心约标签（条件满足时显示）
  - 冲突标记（条件满足时显示）
```

#### 8.3.3 省心约待约卡片

```
类型：sxy-pending
背景：#f6ffed（浅绿色）
边框：1px dashed #b7eb8f（绿色虚线边框）
圆点：#52c41a（绿色）
时间文字：#389e0d
描述文字：#95de64，显示 "{学员姓名} {脱敏手机号}"
右上角：绿色省心约标签（背景 #52c41a）
交互：点击弹出省心约规则详情弹窗
```

### 8.4 卡片内部结构（通用）

```
┌─────────────────────────────────┐
│ ● HH:MM - HH:MM     [标签区域] │
│ 描述文字                        │
└─────────────────────────────────┘

标签区域（右上角，可能同时存在多个）：
  - 省心约标签：绿色圆角矩形，文字"省心约"，9px白色
  - 冲突标记：黄色圆形，文字"!"，16×16px
  - 当两者同时存在时，冲突标记左移（right: 28px）
```

## 十、详情弹窗

### 10.1 通用定位逻辑

```
定位策略：
  1. 优先显示在点击元素右侧（left = rect.right + 8px）
  2. 若右侧空间不足（超出视口），改为左侧（left = rect.left - popWidth - 8px）
  3. 若左侧也不足，水平居中
  4. 纵向：top对齐点击元素顶部，若超出底部则上移
  5. 最小安全边距：10px

关闭方式：
  1. 点击弹窗内关闭按钮（×）
  2. 点击弹窗外部空白区域
  3. 切换视图/翻页/搜索时自动关闭
```

| 当前状态 | "待约课"，绿色(#52c41a)加粗 |

### 10.4 手机号脱敏规则

```
输入：studentId（11位手机号格式）
输出：前3位 + **** + 后4位
示例：15864367001 → 158****7001
异常处理：长度不足7位时返回原值
```

## 十一、省心约规则体系（核心业务逻辑）

### 11.1 概念说明

**省心约**：学员与老师之间约定的固定授课时间。每条规则表示"该学员每周X的某个时间段是固定上课时间"。系统需要检测每个时段的规则执行状态。

### 11.2 数据结构

```
SxyRule {
    studentName: string   // 学员姓名
    studentId: string     // 学员ID（手机号）
    day: number           // 星期几（0=周一 ~ 6=周日）
    start: string         // 开始时间 "HH:MM"
    end: string           // 结束时间 "HH:MM"
}
```

### 11.3 规则状态判定

每个时间槽位根据省心约规则和实际课节的关系，存在以下5种状态：

#### 状态关系矩阵

| # | 条件 | 状态 | 视觉表现 |
|---|------|------|----------|
| 1 | 时段有省心约规则 + 该学员有课节 | **正常命中** | 红色课节卡片 + 右上角绿色「省心约」标签 |
| 2 | 时段有省心约规则 + 无课节（空闲） | **待约课** | 绿色虚线卡片 + 右上角绿色「省心约」标签 |
| 3 | 时段有省心约规则 + 但被**其他学员**约课 | **冲突** | 红色课节卡片 + 右上角黄色「!」标记 |
| 4 | 时段有省心约规则A命中 + 同时有规则B冲突 | **命中+冲突** | 红色课节卡片 + 绿色「省心约」标签 + 黄色「!」标记 |
| 5 | 时段无省心约规则 | **普通课节** | 红色课节卡片，无标签 |

#### 状态1：正常命中（省心约规则 = 是）

```
判定条件（AND关系）：
  sxyRules 中存在规则 r 满足：
    r.day === lesson.day
    AND r.start === lesson.start
    AND r.end === lesson.end
    AND r.studentId === lesson.studentId

含义：该学员在自己的省心约固定时间成功约课
表现：课节卡片右上角显示绿色「省心约」标签
弹窗：省心约规则 = "是"，展示规则时段
```

#### 状态2：待约课

```
判定条件（AND关系）：
  sxyRules 中存在规则 r 满足：
    r.day === currentDay
    AND r.start === slotTime
    AND r.end === slotEndTime
  AND 该时段无任何课节占用

含义：学员的省心约固定时间到了，但课节尚未安排
可能位置：
  a) 在师资固定时间内 → 作为第2层渲染
  b) 不在师资固定时间内 → 作为第3层单独渲染
表现：绿色虚线卡片
弹窗：绿色header，展示省心约规则详情
```

#### 状态3：冲突

```
判定条件（AND关系）：
  sxyRules 中存在规则 r 满足：
    r.day === lesson.day
    AND r.start === lesson.start
    AND r.end === lesson.end
    AND r.studentId !== lesson.studentId

含义：其他学员约了本应属于规则学员的固定时间
表现：课节卡片右上角显示黄色圆形「!」标记
hover提示："省心约冲突：{冲突学员姓名} 的省心约时间"
弹窗：展示冲突警告区域（黄色背景），列出被冲突的学员姓名和ID
```

#### 状态4：命中 + 冲突同时存在

```
发生场景：
  同一时段有多条省心约规则（A学员和B学员都设了这个时间）
  实际约课学员 = A学员
  则：A学员的规则正常命中，B学员的规则被冲突

表现：
  课节卡片同时显示：
    - 绿色「省心约」标签（右上角）
    - 黄色「!」冲突标记（「省心约」标签左侧，right: 28px）
弹窗：同时展示省心约规则=是 + 冲突警告区域
```

### 11.4 省心约待约卡片的渲染位置判定

```
场景A：规则时段在师资固定时间范围内
  判定：fixedSlots.some(([fs, fe]) => ruleStart >= fs && ruleEnd <= fe)
  渲染时机：固定时间段扫描时遇到该规则即渲染
  卡片位置：嵌入固定时间背景中

场景B：规则时段不在师资固定时间范围内
  判定：上述条件不满足
  额外条件：该时段也未被课节占用
  渲染时机：固定时间扫描之后，单独遍历sxyRules
  卡片位置：独立渲染在时间轴上（无蓝色背景）
```

### 11.5 可被预约时间的合并逻辑

在固定时间段内扫描空闲槽位时，连续的"无规则空闲"槽位合并为一个大的"可被预约时间"卡片：

```
合并开始：当前槽位为空闲且无省心约规则
合并结束（遇到以下任一条件时停止）：
  1. 到达固定时间段结束边界
  2. 遇到被课节占用的槽位
  3. 遇到有省心约规则的槽位

合并后渲染为一个卡片：
  时间显示：合并后的完整时间范围
  描述："可被预约时间"
```

## 十二、数据模型

### 12.1 Teacher（师资）

```
Teacher {
    name: string          // 师资姓名
    id: string            // 师资ID
    fixed: Fixed[]        // 固定授课时间段列表
    sxyRules: SxyRule[]   // 省心约规则列表
    lessons: Lesson[]     // 已约课节列表
}
```

### 12.2 Fixed（固定时间段）

```
Fixed {
    day: number           // 星期几（0-6）
    start: string         // 开始时间 "HH:MM"
    end: string           // 结束时间 "HH:MM"
}
```

说明：同一天可以有多个不连续的固定时间段（如 14:00-16:00 和 17:00-21:00），分别渲染为独立的蓝色背景块。

### 12.3 SxyRule（省心约规则）

```
SxyRule {
    studentName: string   // 学员姓名
    studentId: string     // 学员ID（手机号）
    day: number           // 星期几（0-6）
    start: string         // 开始时间 "HH:MM"
    end: string           // 结束时间 "HH:MM"
}
```

### 12.4 Lesson（课节）

```
Lesson {
    id: string            // 课节ID（如 SXY20260610001、OES80001）
    type: string          // 来源类型："sxy" 或 "oes"（仅数据标识，UI不区分）
    day: number           // 星期几（0-6）
    start: string         // 开始时间 "HH:MM"
    end: string           // 结束时间 "HH:MM"
    studentName: string   // 学员姓名
    studentId: string     // 学员ID（手机号）
    tutor: string         // 二讲老师姓名
    tutorId: string       // 二讲老师ID
    courseType: string     // 课程类型（如"阶段学-补差距"、"同步学"）
    className: string     // 课节名称
    classId: string       // 班级ID
    status: string        // 课节状态（如"未开课"）
}
```

### 12.5 StudentEndDates（学员预计结课时间）

```
StudentEndDates: Map<studentId, dateString>
// key: 学员ID
```

## 十五、边界情况与异常处理

### 15.1 课节在非固定时间内

课节可以存在于师资固定时间之外（如22:00-22:30）。此类课节正常渲染，但没有蓝色背景。

### 15.2 省心约规则在非固定时间内

省心约规则的时段也可以不在师资固定时间范围内。待约状态下正常渲染绿色虚线卡片（无蓝色背景）。

### 15.3 同一时段多条省心约规则

同一时段可以有多个学员的省心约规则。如果一个学员约了课，其他学员的规则均为冲突状态。

### 15.4 预计结课时间缺失

如果 studentEndDates 中没有该学员的数据，显示"—"。

### 15.5 师资无数据

如果师资没有 sxyRules，省心约相关的所有逻辑（标签、冲突、待约）均不执行。
